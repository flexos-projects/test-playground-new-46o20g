<flex_block type="instructions">
Reference docs are SDK/framework documentation for the builder AI.
These are imported or generated based on the project's technical specs.

Each file covers one technology: its key APIs, patterns, gotchas,
and best practices relevant to this project. These are NOT specs —
they don't define what to build. They tell the builder HOW to use
the tools that technical specs have chosen.

Generated from: scraping docs sites, extracting relevant sections,
tailoring to this project's specific usage patterns.

Regenerated when: technical specs change (e.g., switching from
Firebase to Supabase triggers new reference docs).

NAMING: NNN-{slug}.md
EXAMPLE: 001-nuxt.md, 002-firebase.md, 003-stripe.md

More reference docs than needed is fine — the builder AI reads what's
relevant. Having outdated reference docs for a removed integration
is low-cost. Regenerate when technical specs change.
</flex_block>
