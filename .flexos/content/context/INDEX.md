<flex_block type="instructions">
The context collection is the default catch-all for information that
doesn't fit a structured collection. Drop anything here:
- Company history and mission
- Team bios
- Brand voice guidelines
- Competitor research
- Marketing copy
- Client feedback
- Random notes and context

The AI reads these when generating specs, prototypes, and builds
to stay aligned with the project's identity and requirements.
</flex_block>

# Context Collection

This collection stores unstructured project knowledge. Each file is a
topic with freeform markdown content. No schema — just prose.
