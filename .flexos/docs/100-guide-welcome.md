# Welcome to FlexOS

FlexOS organises your project as a collection of spec files. Each spec type
has its own folder and panel in the sidebar.

## How to Work

1. **Talk to the AI** — describe what you want to build
2. **Specs appear** — the AI creates numbered spec files in the right folders
3. **Core docs update** — holistic summaries regenerate as specs evolve
4. **Prototypes generate** — visual HTML previews of your pages
5. **Build executes** — AI reads everything and produces production code

## Spec Types

- **Vision** — goals, constraints, audience, brand identity
- **Features** — what your app does (primary and secondary features)
- **Pages** — routes, components, layouts, overlays
- **Flows** — user journeys, auth flows, backend processes
- **Database** — collections, APIs, stores
- **Design** — tokens (colors, typography), patterns (CSS), reference (brand guidelines)
- **Technical** — stack decisions, integrations, infrastructure, auth
- **Content** — collection definitions and seed records

## Tips

- Everything is numbered in the order it was created
- Every file has frontmatter (metadata) and optional flex_blocks (structured data)
- Use Spaces to organise focused conversations
- All changes are tracked in Commits automatically
